Git-Workflow

Mit dem Befehl git clone wird aus dem git repository die Projektdatei(n) auf der lokalen Festplate kopiert, damit man die Daten lokal - auf eigene Festplatte - bearbeiten kann. Wenn man die Datei(n) bearbeitet/ verändert hat, dann kann man die veränderte Datei mit dem Befehl git add für die Zwischenspeicherung vermerken. Mit dem weiteren Befehl git commit wird ein Kommentar bezüglich der Bearbeitung von Datei(n)hinterlegt sowie die veränderte Datei(n) lokal zwischengespeichert. Mit dem Befehl git push werden die veränderte Datei(n) auf das git repository hochgeladen. Desweiteren kann man mit dem Befehl git pull die lokalen Projektdateien auf den Stand der Dateien im git repository aktualisieren.

Kopiere den Link "https://github.com/Aguona/EWP-HA-16.git"
Gib im Terminal ein: git add Projekt.html
Gib im Terminal ein: git commit -m "Neue Tabelle (Nr. 13) hinzugefügt"
Gib im Terminal ein: git push
Gib im Terminal ein: git pull
